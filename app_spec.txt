<overview>
# New API — macOS Vibrancy UI Redesign

New API 是一个 Go 语言后端 + React 前端的 API 管理平台，用于管理 AI 模型渠道、令牌、用户、
订阅、计费、日志和 Playground 调试。当前项目已有完整功能，本次任务是对整个前端 UI 进行
系统性的 macOS Vibrancy 风格重构，覆盖所有页面和组件。

目标：将所有 UI 组件统一为 macOS 原生暗色毛玻璃风格（macOS Vibrancy），提升视觉一致性、
信息层级清晰度和交互品质，同时保证所有已有功能正常运行。

## 项目路径
- 前端代码：`/data/dev/go/new-api/web/`
- 前端源码：`/data/dev/go/new-api/web/src/`
- 后端代码：`/data/dev/go/new-api/`（Go，不改动后端逻辑）

## 核心原则
1. **只改 UI，不改功能**：后端 API 不动，前端业务逻辑不动，只改样式/布局/组件外观
2. **macOS Vibrancy 风格全面贯穿**：三层深灰系统、毛玻璃效果、克制装饰、系统级质感
3. **每个组件都必须优化**：不能遗漏任何组件，包括 modals、tooltips、dropdowns、tags、badges
4. **保持响应式**：桌面端和移动端都要适配
5. **保持 i18n**：不破坏现有的 7 语言国际化支持
6. **保持暗色/亮色/自动三模式**：已有 ThemeProvider 支持 light/dark/auto
</overview>

## 设计系统 — macOS Vibrancy（毛玻璃原生风格）

### 设计哲学
macOS Vibrancy 是 Apple 桌面应用设计语言的 Web 还原。核心不是炫技，而是**克制**。
通过多层深灰面板、系统级 backdrop-blur 和极度克制的装饰，还原桌面应用的沉稳质感。

设计隐喻：**macOS 系统偏好设置** — 每个面板有明确的深度层级，信息通过位置和
背景色差传达，而非花哨的装饰。

核心特征：**Restraint（克制）** — 没有渐变、没有大阴影、没有圆角过度、
没有装饰性动画。深度来自背景层级，交互来自微妙的颜色过渡。

### 圆角系统（macOS 级 — 最大 rounded-xl/12px）
- sm: 6px (--radius-sm) → tags, badges, navigation items, inputs, buttons, selects
- md: 8px (--radius-md) → inputs, buttons, dropdowns, popovers, tabs
- lg: 12px (--radius-lg) → cards, modals, panels
- **禁止**: rounded-3xl, rounded-full（除头像外）, 任何大于 12px 的圆角
- **已在 index.css 全局覆盖**: .semi-tag → radius-sm, .semi-input-wrapper → radius-md, .semi-card → radius-lg, .semi-modal-content → radius-lg

### 阴影系统（极简 — 深度来自背景色差）
- sm: none (--shadow-sm)
- md: 0 1px 0 rgba(0,0,0,0.05) / 0 1px 0 rgba(255,255,255,0.05)
- float: 0 2px 8px rgba(0,0,0,0.12) / 0 2px 8px rgba(0,0,0,0.3)
- ring: 0 0 0 0.5px rgba(0,0,0,0.06) / 0 0 0 0.5px rgba(255,255,255,0.06)
- **禁止**: shadow-xl, shadow-2xl, shadow-lg, 任何大扩散阴影
- Tailwind 预设: mv-xs, mv-sm, mv-md, mv-float, mv-ring

### 字体
- 正文: Inter, -apple-system, system-ui, sans-serif (--font-sans)
- 标题: Georgia, "Times New Roman", serif (--font-serif) — 标题使用衬线体
- 代码: "JetBrains Mono", "SF Mono", ui-monospace (--font-mono)
- 正文字号: 14px (text-sm)
- 标题文本: text-white/95 (暗色) / text-black/85 (亮色)
- 正文文本: text-white/70 (暗色) / text-black/55 (亮色)

### 间距
- Section: py-12 md:py-16
- Container: px-4 md:px-6
- Card 内边距: p-4 md:p-6
- 元素间距: 使用 Tailwind gap 工具类

### 动效系统（极简 — 仅颜色过渡）
- micro: 150ms (--ease-micro) → 按钮 hover、输入框 focus
- normal: 280ms (--ease-normal) → 面板展开、主题切换
- slow: 500ms (--ease-slow) → 页面级过渡
- 缓动: cubic-bezier(0.2, 0, 0, 1) (mv-ease)
- 过渡属性: 只做 color, background-color, border-color, opacity
- **禁止**: bounce, elastic, spring, pulse, spin, 任何装饰性动画
- **禁止**: bg-gradient-to-r/br/l 等渐变背景
- 可接受: 加载状态的淡入淡出 (mv-fade-in: translateY(8px) + opacity)

### 毛玻璃效果（核心视觉特征）
- 头部导航: glass-header → backdrop-blur(20px) + 半透明背景 + 底部 1px 边框
- 侧边栏: glass-sidebar → backdrop-blur(20px) + 半透明背景 + 右侧 1px 边框
- 浮动面板: glass-float → backdrop-blur(16px) + 1px 全边框
- 下拉菜单: 同 glass-float
- 弹窗遮罩: rgba(0,0,0,0.4) + backdrop-blur(4px)
- **注意**: 侧边栏暗色模式使用 bg-[#1c1c1e]/80 实现 vibrancy

### 组件层级深度（通过背景色区分）
```
Level 0: --bg-base (#F5F5F7 / #1c1c1e)     → 页面背景
Level 1: --bg-subtle (#EDEDF0 / #252527)    → 侧边栏/次级区域
Level 2: --surface (#FFFFFF / #2c2c2e)      → 卡片/面板
Level 3: --surface-active (/ #3a3a3c)       → 提升元素/hover态/按钮
```

<coding_guidelines>

## React + Semi Design + Tailwind 编码规范（强制执行）

### 1. 样式优先级
- **CSS Variables (--xxx)** 是设计令牌的唯一来源
- **Tailwind 工具类** 用于布局、间距、响应式
- **Semi Design 组件** 已通过 index.css 全局覆盖样式
- 行内 style 仅在动态值时使用
- **禁止**: 在组件中硬编码颜色值，必须使用 CSS 变量或 Tailwind mv-* 类

### 2. Semi Design 组件使用
- 优先使用 Semi Design 组件（Button, Input, Select, Table, Modal, Card, Tag, etc.）
- 所有 Semi 组件已通过 CSS 全局覆盖继承 Vibrancy 令牌
- 如需自定义 Semi 组件样式，在 index.css 添加全局覆盖，不要在组件内 !important
- 使用 Semi 的 theme-mode="dark" 属性支持暗色模式

### 3. CSS 变量使用规范
```css
/* 正确 — 使用设计令牌 */
background: var(--surface);
color: var(--text-primary);
border: 1px solid var(--border-default);

/* 正确 — 使用 Tailwind mv-* 类 */
className="bg-mv-surface text-mv-accent border-mv-bg-subtle"

/* 错误 — 硬编码颜色 */
background: #2c2c2e;
color: white;
```

### 4. 暗色模式适配
- 所有颜色必须通过 CSS 变量，变量已在 :root 和 html.dark 中定义
- 不要用 Tailwind dark: 前缀覆盖颜色 — CSS 变量自动切换
- 测试时必须在 light、dark、auto 三种模式下验证

### 5. 毛玻璃效果
- 使用预定义的 glass-* 工具类：glass-header, glass-sidebar, glass-float
- 自定义毛玻璃: `backdrop-filter: blur(Npx); -webkit-backdrop-filter: blur(Npx);`
- 必须包含 -webkit- 前缀（Safari 兼容）

### 6. 组件布局
- 卡片: 使用 mv-card 类或 `bg-[var(--surface)] border border-[var(--border-default)] rounded-[var(--radius-lg)]`
- 表格: 无交替行色，仅用 border-subtle 分隔行
- 标签: rounded-[var(--radius-sm)]，小巧精致
- 按钮: 暗色模式下 bg-[#3a3a3c] text-white/90，亮色使用 Semi 默认
- 输入框: 暗色模式下 bg-[#1c1c1e] border-white/10

### 7. 禁止模式（Forbidden Patterns）
以下 CSS 类/模式绝对不能出现：
- `bg-gradient-to-r`, `bg-gradient-to-br`, `bg-gradient-to-l` → 渐变背景
- `shadow-xl`, `shadow-2xl`, `shadow-lg` → 大阴影
- `rounded-3xl`, `rounded-full`（头像除外） → 过度圆角
- `animate-*`（bounce, pulse, spin 等） → 装饰性动画
- `border-2`, `border-3`, `border-4` → 粗边框（只用 1px）
- 在 JSX 中写死颜色十六进制值 → 必须用 CSS 变量

### 8. 响应式设计
- 移动端 (<768px): 侧边栏收起为抽屉，表格使用卡片视图
- 桌面端 (>=768px): 侧边栏固定，表格使用标准列视图
- 使用 useIsMobile() hook 检测设备
- 表格支持 compact mode（useTableCompactMode）

### 9. 图标系统
- 主要: @douyinfe/semi-icons
- 补充: lucide-react
- 补充: react-icons
- 图标颜色跟随 CSS 变量 (text-primary, text-secondary, text-muted)

### 10. 状态管理
- React Context: UserContext, StatusContext, ThemeContext, PlaygroundContext
- 自定义 hooks: useChannelsData, useUsersData, useModelsData, etc.
- 不要改变状态管理模式，只改 UI 渲染部分

</coding_guidelines>

## 组件设计规格（macOS Vibrancy 标准）

### 导航侧边栏 (SiderBar)
- 背景: glass-sidebar (backdrop-blur(20px) + 半透明)
- 宽度: 展开 180px, 收起 60px (CSS 变量 --sidebar-width)
- 导航项: rounded-[var(--radius-sm)], margin 2px 8px
- 选中态: bg accent-light + 左侧 3px accent 指示条
- hover: bg surface-hover
- 图标: 22px container, 10px margin-right
- 分组标签: 11px, uppercase, letter-spacing 0.5px, text-muted
- 底部折叠按钮: sticky, glass-bg + 顶部 1px border-subtle
- 过渡: width 0.3s ease

### 头部导航 (HeaderBar)
- 背景: glass-header (backdrop-blur(20px) + 底部 1px border)
- 高度: 64px
- Logo: 左侧固定
- 导航: 水平菜单
- 右侧: 通知 + 语言 + 主题切换 + 用户菜单
- 所有按钮: 仅图标, hover 时微妙背景色变化

### 卡片 (Card)
- 背景: var(--surface)
- 边框: 1px solid var(--border-subtle)
- 圆角: var(--radius-lg) = 12px
- 阴影: none（已全局覆盖 .semi-card）
- 内边距: header 16px 20px, body 20px
- hover: 仅 border-color 变化，无 scale/shadow 动画

### 表格 (Table)
- 无交替行色 — 干净、统一
- 行分隔: 1px solid var(--border-subtle)
- hover: rgba(255,255,255,0.04) (暗色)
- 表头: 与内容同背景，不加重
- 滚动条: 6px, semi-grey-2 @ 0.3 opacity
- Compact mode: 通过 useTableCompactMode 控制

### 按钮 (Button)
- Primary: bg-[#3a3a3c] text-white/90 (暗色) / Semi 默认 (亮色)
- Accent: bg-[#0a84ff] text-white hover:bg-[#0a84ff]/85
- Outline: bg-transparent border-white/12 text-white/70 hover:bg-white/5
- 圆角: var(--radius-md) = 8px
- 过渡: transition-colors duration-200
- 禁止: 任何 scale 动画、任何 shadow 效果

### 输入框 (Input)
- 暗色: bg-[#1c1c1e] border-white/10
- 亮色: Semi 默认
- 圆角: var(--radius-md) = 8px
- placeholder: text-white/30 (暗色)
- focus: outline-none, border-white/25
- 过渡: transition-colors duration-200

### 标签 (Tag)
- 圆角: var(--radius-sm) = 6px
- 小巧精致，贴合内容
- 颜色: 使用 Semi 语义色 (primary, success, warning, danger)
- 可使用 sbg-variant-* 色彩变体 (violet, teal, amber, rose, green)

### 弹窗 (Modal)
- 遮罩: rgba(0,0,0,0.4) + backdrop-blur(4px)
- 内容: bg var(--surface) + 1px border-default + var(--shadow-float)
- 圆角: var(--radius-lg) = 12px
- 动画: 仅 fade-in (opacity), 不要 scale

### 下拉菜单 (Dropdown/Popover)
- 背景: glass-bg (毛玻璃)
- 边框: 1px glass-border
- 圆角: var(--radius-md) = 8px
- 阴影: var(--shadow-float)
- 菜单项 hover: bg rgba(255,255,255,0.08)

### Tooltip
- 背景: #3a3a3c
- 边框: 1px rgba(255,255,255,0.1)
- 圆角: var(--radius-sm)

### 分割线 (Divider)
- 颜色: var(--border-subtle)
- 极淡，几乎不可见 — 用留白代替分割线

### Tab 导航
- ink-bar: 2px 圆角, 200ms ease-out 过渡
- 内容区: padding 0, flex 布局

### 图表 (Charts - VChart)
- 背景: 透明或 var(--surface)
- 配色: 使用 iOS 色系 (accent, success, warning, error, info)
- 无网格线或极淡网格线
- tooltip: glass-float 风格

### 骨架屏/加载
- 使用 mv-fade-in 动画进入
- 加载器: macOS 风格旋转条（mv-loader）— 12 条辐射线
- 不使用 spin 或 pulse 动画

### 空状态
- 居中图标 + 说明文字
- 图标颜色: text-muted
- 文字颜色: text-secondary

### 搜索
- 输入框: 标准 input 样式
- 搜索弹窗: glass-float 风格
- 结果列表: 简洁无装饰

### 通知
- 使用 Semi Toast/Notification
- 样式跟随全局 Vibrancy 覆盖

## 实施方法

### 分阶段推进
1. **设计基础 (P1)**: 完善 CSS 变量、Tailwind 配置、Semi 全局覆盖、index.css
2. **布局骨架 (P1)**: PageLayout, HeaderBar, SiderBar, Footer
3. **通用组件 (P1)**: Card, Loading, Modal, Tag, Button, Input, Table 全局样式
4. **核心页面 (P2)**: Dashboard, Playground, 各管理页面
5. **表格系统 (P2)**: 所有 CRUD 表格、过滤器、操作栏、弹窗
6. **设置系统 (P2)**: 所有设置页面和子组件
7. **辅助页面 (P3)**: Auth, Setup, TopUp, About, NotFound, etc.
8. **精细打磨 (P3)**: Markdown 样式、图表配色、动画过渡、响应式微调
9. **验证测试 (P3)**: 全页面功能验证、暗色/亮色/自动模式测试

### 每个组件的优化清单
对于每个组件，需要检查并优化：
- [ ] 背景色是否使用 CSS 变量
- [ ] 文字颜色是否使用 CSS 变量
- [ ] 边框是否为 1px + CSS 变量色
- [ ] 圆角是否符合 macOS 规范 (sm/md/lg)
- [ ] 阴影是否极简或无
- [ ] 间距是否合理
- [ ] hover/active 状态是否仅有颜色过渡
- [ ] 暗色模式是否正常
- [ ] 移动端适配是否正常
- [ ] 无禁止模式 (渐变/大阴影/大圆角/动画)
- [ ] 毛玻璃效果是否正确使用
- [ ] 组件大小/位置是否合理
- [ ] i18n 文本是否完整
